# Security Policy

## Supported Versions

The following versions of `ccc` (Copilot Config CLI) currently receive security updates:

| Version | Supported          |
| ------- | ------------------ |
| 0.x     | :white_check_mark: |

## Reporting a Vulnerability

If you discover a security vulnerability in `ccc`, please report it responsibly using **GitHub's private vulnerability reporting** feature:

1. Navigate to [jsburckhardt/co-config — Security Advisories](https://github.com/jsburckhardt/co-config/security/advisories/new).
2. Click **"Report a vulnerability"**.
3. Provide a clear description of the vulnerability, steps to reproduce, and any potential impact.

> **Please do not open public issues for security vulnerabilities.** Private reporting ensures that vulnerabilities can be addressed before they are publicly disclosed.

We will acknowledge your report within **5 business days** and aim to provide an initial assessment (accepted, declined, or needs more information) within **10 business days**.

## Disclosure Policy

We follow a **coordinated disclosure** process:

1. **Fix first** — once a vulnerability is confirmed, we develop and test a fix before any public disclosure.
2. **Release** — a patched version is published to GitHub Releases.
3. **Disclose** — a public security advisory is issued describing the vulnerability, its impact, and the fix.
4. **Credit** — reporters are credited in the advisory unless they wish to remain anonymous.

We appreciate the security community's efforts in responsibly disclosing vulnerabilities and helping keep `ccc` safe for everyone.
